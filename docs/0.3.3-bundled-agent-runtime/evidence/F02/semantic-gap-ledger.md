## Critical gap rule

Any `open-critical` gap remains `UNKNOWN_CRITICAL` or `SEMANTIC_BLOCKED`; it cannot be downgraded because a JSON field happens to have the same name. Current critical gaps are F02-G01, F02-G06, F02-G07, F02-G08, F02-G13, and F02-G15.

## Scope assignment

- F01: source closure, immutable source identity, forbidden imports and config discovery.
- B02: model/context/budget foundation.
- B03: grant/capability and skill/hook boundary.
- F04: deterministic vertical trace with one tool, permission denial, cancellation race, compact/resume checkpoint, and unknown-event fail-closed behavior.
