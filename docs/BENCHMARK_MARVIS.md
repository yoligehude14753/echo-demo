# EchoDesk 对标 Marvis · 定位分析与体验优化清单

> 记录时间：2026-06-02
> 对标对象：[腾讯 Marvis（马维斯）](https://marvis.qq.com/)
> 本文用途：①厘清 EchoDesk 与 Marvis 的定位关系；②沉淀 EchoDesk 当前"还没做到丝滑"的体验欠缺，作为**持续优化清单**；③给出差异化路线。

---

## 1. 一个关键判断：不是同一类产品

| | EchoDesk | Marvis |
|---|---|---|
| 本质 | **持续陪伴 + 声音交互**的个人数字分身 | **操作系统层级 AI 助手**（电脑管家 / 任务执行）|
| 核心动作 | 被动持续听 → 记忆 → 语音问答/产物 | 主动下指令 → 操控系统/文件/跨端 |
| 交互重心 | 声音优先、免手、长期记忆 | 任务指令、系统闭环操控 |

**结论**：EchoDesk 的"持续陪伴 + 声音交互"恰恰是 Marvis **不做**的方向。我们感受到的"差距"主要是两块——
1. **体验打磨**（稳定性 / 延迟 / 流畅度）← 这是要持续优化的重点
2. **能力广度**（Marvis 的系统操控 / 文件图库 / 跨端那一摊我们没有，但那不是陪伴方向的必需项）

不应去追 Marvis 的"电脑管家"，而要把"陪伴 + 声音"做到 Marvis 做不到的深度。

## 3. 持续优化清单（"还没达到丝滑"的体验欠缺）

> 这是我要**持续优化**的核心 backlog。按对"陪伴 + 声音"体验的影响排优先级。

### P0 · 语音链路丝滑度（最影响陪伴体感）
- **现状**：已从固定 4s 硬切改为 VAD 断句 + 静音 endpoint 自适应收尾；唤醒词换成音节级模糊匹配 + 跨 chunk 拼接。
- **还差**：
  - 唤醒成功率 / 误唤醒率没有量化基线，需真人多场景实测
  - 断句时机（说完判定）在"长指令带思考停顿"下仍可能偏早/偏晚，需实测调参
  - 没有"打断 / 连续对话 / 多轮免唤醒续聊"——离"像打电话一样自然"还有距离
- **目标**：唤醒→说→答→朗读一气呵成；支持打断、连续多轮、低误触。

### P0 · TTS 可靠性与即时性
- **现状**：句级流式播放（前瞻消除句间空档）+ instructions 抑制唱歌；动态判 enabled。
- **还差**：偶发无声/不稳定需彻底坐实；首句起播延迟仍受云端合成影响。
- **目标**：开了就一定响、首句尽快出声、不乱唱、可随时中止。

### P1 · 声纹准确度
- **现状**：阈值 0.55→0.50、ambient 上限 6→4 缓解"2 人识别成 4 人"。
- **还差**：无 enrollment 的 ambient 声纹本就难；需"会议内声纹绑定 + 用户改名长期记忆 + 同人合并"。
- **目标**：常见 2-4 人小场景说话人数量稳定、可命名、跨会议复用。

### P1 · 端到端延迟体感
- **现状**：STT / LLM / TTS 全云端串联，首字慢。
- **还差**：缺端云双模式；缺本地小模型兜底（唤醒/短问答）。
- **目标**：参考 Marvis 隐私模式，本地兜底降低首字延迟，断网也能基本对话。

### P1 · 智能对话稳定性
- **现状**：agent 三级兜底（永不硬失败）+ 单步重试 + 重复调用拦截 + 显式产物直连生成。
- **还差**：单循环编排在复杂任务下仍易抖动；可考虑借鉴专职 Agent 分工。
- **目标**：复合任务（调研→生成）稳定闭环，过程可视、可中止。

### P2 · 产物视觉与自适应
- **现状**：PPT 对齐 N-v3 投行风、Word 标书形式、内容自适应。
- **还差**：模板风格单一；图表/配图能力弱。
- **目标**：更多体裁模板 + 数据图表 + 配图。

## 4. 差异化路线（把 Marvis 不碰的地方做透）

1. **被动记忆 → 主动陪伴**：不止"记下来能查"，要做主动提醒 / 今日回顾 / 待办催办 / 主动总结——这才是"陪伴"。
2. **声音作为第一交互**：唤醒→说→答→朗读做到打电话般自然（打断、连续对话、低延迟）。
3. **关系与人设记忆**：记住"谁是谁、聊过什么、偏好"，长期形成"懂你的分身"。
4. **端侧低延迟兜底**：本地小模型做唤醒/短问答，降低首字延迟，兼顾隐私。

---

## 附：与 Marvis 的边界（明确不做）

- 不做 OS 系统操控（改设置、清文件、关广告）——那是 Marvis 的主场。
- 不做跨端操控手机 App。
- 文件管理只服务于"记忆与检索"，不做通用文件管理器/AI 图库。
